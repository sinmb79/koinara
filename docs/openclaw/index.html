ROLE SPLIT

What OpenClaw does, and what it does not do.

OpenClaw can do

Prompt planning, operator workflow, response drafting, agent-to-agent coordination, and human-in-the-loop review.

Koinara-node must do

Job polling, manifest and receipt checks, response hash submission, verifier voting, PoI finalization, and reward settlement.

OpenClaw alone cannot do

OpenClaw by itself is not a provider or verifier node. Without a running node, it does not participate on-chain.

CURRENT STATE

Built-in backend and reusable skill package are both available now.

What exists now

OpenClaw can now be used directly as a built-in provider backend inside Koinara-node, and the repo ships a reusable OpenClaw skill package.

What still does not exist yet

A first-class built-in command like openclaw connect koinara-node is still not implemented.

Clean future direction

The next clean step is a richer skill package that can talk to a local or remote Koinara-node with more automation.

REWARD FLOW

How an OpenClaw-backed provider actually earns.

1. OpenClaw drafts the response

The node passes the manifest prompt into the local OpenClaw CLI. OpenClaw acts as the response engine, not the chain participant.

2. Koinara-node submits and settles

Koinara-node hashes the response, submits it on-chain, handles verifier participation, and keeps heartbeat plus epoch state.

3. Claim after the epoch closes

Protocol rewards are minted only when the node wallet sends the claim transaction after the current epoch has closed.

npm run provider:v2:openclaw:start
npm run provider:v2:openclaw:claim
npm run verifier:v2:openclaw:start
npm run verifier:v2:openclaw:claim

RECOMMENDED SHAPE

Use the agent above the node.

Recommended

OpenClaw for orchestration, Koinara-node for execution

Let OpenClaw prepare prompts, manage sessions, and assist with review. Let Koinara-node keep control of wallet signing, contract interaction, duplicate suppression, crash recovery, and final protocol participation.

Provider backend

OpenClaw can now generate provider output

Set provider.backend = "openclaw" and the node will pass each manifest prompt through the local OpenClaw CLI before hashing and submitting the response.

Operator pattern

Single machine

Run OpenClaw and Koinara-node on the same machine when you want local agent assistance with minimal setup.

Operator pattern

Two-process split

Keep OpenClaw as the interactive process and let the node run as the long-lived daemon that actually participates on-chain.

Future

Dedicated skill package

The repo now ships the first bundled OpenClaw skill. The long-term path is a richer package that can talk to a local or remote node runtime.

OpenClaw improves the workflow. Koinara-node remains the protocol execution boundary.
