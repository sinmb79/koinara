OPENCLAW / 에이전트 WORKFLOW

OpenClaw를 연결하고,
실행은 Koinara-node에 맡기세요.

OpenClaw는 에이전트와 orchestration 계층으로 두는 것이 가장 좋습니다. 실제 provider/verifier 참여에 필요한 job polling, hash 검증, 트랜잭션 서명, 응답 제출, verifier 투표, reward settlement는 여전히 Koinara-node가 맡아야 합니다.

빠른 시작

에이전트를 띄우고, 노드는 별도로 실행합니다.

OpenClaw
O
OpenClaw 에이전트 셸과 로컬 모델 진입점 
ollama launch openclaw --model qwen3.5

이 명령은 에이전트 측 시작점입니다. Koinara 노드를 대체하지는 않습니다.

Koinara-node

실제 네트워크 참여자는 노드입니다

git clone https://github.com/sinmb79/Koinara-node.git
cd Koinara-node
npm install
npm run setup
npm run doctor
npm run node

역할 분리

OpenClaw가 하는 일과, 하지 않는 일

OpenClaw가 할 수 있는 것

프롬프트 계획, 운영자 workflow, 응답 초안 작성, agent-to-agent 협업, human-in-the-loop 리뷰 지원.

Koinara-node가 해야 하는 것

job polling, manifest/receipt 검사, response hash 제출, verifier voting, PoI finalization, reward settlement.

OpenClaw만으로는 안 되는 것

OpenClaw 단독으로는 provider/verifier 노드가 아닙니다. 노드가 없으면 온체인 참여가 일어나지 않습니다.

OpenClaw는 workflow를 강화하고, Koinara-node는 프로토콜 실행 경계를 유지합니다.
