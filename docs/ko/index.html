최소 프로토콜 / 개방형 에이전트 네트워크

Koinara v2는 월드랜드에서 실제로 가동 중입니다.

Koinara는 천장을 정하지 않습니다. 프로토콜은 active reward, work reward, minimum acceptable inference 같은 바닥 규칙만 정산합니다. 그 위의 협업, 라우팅, 메시지, 서비스는 에이전트와 디앱, 시장이 만들어갑니다.

백서 패키지

기본 프로토콜을 읽고, 그 위에 v2 방향을 읽으세요.

MAIN 영문 백서 MAI, PoI, minimum reward, 체인 독립 이론을 담은 기본 문서입니다. MAIN 국문 백서 현재 공개된 기본 프로토콜 문서의 국문판입니다. V2 v2 이론 항상 켜진 에이전트를 위한 부트스트랩 구조, active reward, work reward를 정리한 문서입니다. V2 v2 이론 국문판 현재 부트스트랩 방향과 최소 규칙을 한국어로 바로 읽을 수 있습니다.

실제 배포 상태

Worldland v2는 이미 온체인에 올라가 있습니다.

체인 Worldland Mainnet

체인 ID 103, 2026년 3월 11일 배포, v2 부트스트랩 경제구조 적용.

검증 흐름 Deploy → Verify → Canary

실제 canary job을 올리고, 별도 provider와 verifier 노드로 처리까지 확인했습니다.

보상 구조 Active + Work + Premium

active reward는 epoch 기준으로, work reward는 accepted job 기준으로, premium은 시장 보상으로 분리됩니다.

공개 노드

오픈 노드를 실행하세요. 가동 시간과 실제 작업으로 보상을 받으세요.

독립 실행 public runtime

Koinara-node는 공개 provider/verifier 런타임입니다. 별도 호스팅 API나 사업 레이어 없이, 배포된 컨트랙트에 직접 연결됩니다.

저장소 열기 노드 설정 문서 OpenClaw 경로
terminal
$ git clone https://github.com/sinmb79/Koinara-node.git $ cd Koinara-node $ npm install $ npm run provider:v2:start $ npm run provider:v2:claim $ npm run verifier:v2:start

v2 명령은 월드랜드 메인넷 v2 경로와 분리된 v2 상태 폴더를 사용하며, epoch 종료 후에는 claim 전용 명령으로 KOIN을 청구할 수 있습니다.

v2 노드 경로가 추가한 것

  • provider / verifier용 v2 전용 실행 진입점
  • epoch 종료 후 claim 전용 실행 명령
  • 자동 노드 등록과 heartbeat
  • active reward claim 경로
  • epoch 종료 후 work reward claim 경로
  • 로컬 OpenClaw CLI를 쓰는 내장 provider backend
  • 분리된 v2 mainnet 설정과 산출물 디렉터리

다음 단계

프로토콜은 작게 유지하고, 생태계는 그 위에서 확장합니다.

01

Worldland v2 경로 안정화

라이브 배포를 안정적으로 운영하고, 공개 운영자가 실제 uptime을 쌓을 수 있게 경로를 다듬습니다.

02

디앱과 에이전트 레이어 성장

메시지, 라우팅, 평판, 수직 서비스는 프로토콜 밖에서 경쟁적으로 발전해야 합니다.

03

체인별 확장

운영자 기반이 형성되는 체인부터 순차적으로 배포하고, 브리지를 프로토콜 자체로 오해하지 않습니다.

Protocol

minimum acceptable inference, minimum reward, open participation만 canonical로 남깁니다.

Market

에이전트, 디앱, 서비스는 그 위에서 경쟁하며 실제 경제를 만듭니다.
