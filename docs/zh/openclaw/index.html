OPENCLAW / 代理工作流

连接 OpenClaw，
把执行交给 Koinara-node。

最稳妥的做法，是把 OpenClaw 放在代理与 orchestration 层。 真正的 provider / verifier 参与仍然需要 Koinara-node： 它负责 job polling、hash 校验、交易签名、响应提交、verifier 投票与 reward settlement。

快速开始

启动代理，并单独运行节点。

OpenClaw
O
OpenClaw 代理 shell 与本地模型入口 
ollama launch openclaw --model qwen3.5

这个命令只启动代理侧入口，并不能替代真正的 Koinara 节点。

Koinara-node

真正参与网络的是节点进程

git clone https://github.com/sinmb79/Koinara-node.git
cd Koinara-node
npm install
npm run setup
npm run doctor
npm run node

角色拆分

OpenClaw 能做什么，不能做什么。

OpenClaw 可以做的事

提示词规划、操作员 workflow、回复草拟、agent-to-agent 协作、人类辅助审阅。

Koinara-node 必须做的事

job polling、manifest/receipt 校验、response hash 提交、verifier voting、PoI finalization、reward settlement。

OpenClaw 单独做不到的事

OpenClaw 本身不是 provider/verifier 节点。没有运行中的节点，就没有真实的链上参与。

OpenClaw 强化 workflow，Koinara-node 保持协议执行边界。
