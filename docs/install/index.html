DOCS / WINDOWS INSTALL

Install Koinara-node from a clean Windows shell.

This page is the exact Windows path for a first-time operator. It avoids the common PowerShell npm.ps1 error, keeps the repo under the current user folder, and uses the live Worldland v2 commands.

Start with step 1 OpenClaw guide Repo doc

AGENT HANDOFF

You can hand this guide to your OpenClaw agent.

The intended early operator path is Worldland + OpenClaw + Koinara. That means you do not need to manually type every shell command yourself. Give the GitHub install document to your OpenClaw agent and let it do the repetitive node work. 

Use this guide as the exact installation and connection checklist:
https://github.com/sinmb79/Koinara-node/blob/main/docs/install-windows.md

Install Koinara-node under %USERPROFILE%\koinara-node, run setup, switch the v2 runtime to the local Worldland RPC if available, run openclaw:connect, run provider:v2:openclaw:check, then run provider:v2:openclaw:start. After each stage, tell me what is connected, which RPC is being used, what the current epoch is, and when rewards become claimable.

FLOW

Follow the steps in order.

Step 1

Clone under your user folder

Do not clone into Desktop or system32.

 Step 2

Run setup and accept the home-path defaults

Runtime files now live under %USERPROFILE%\.koinara-node.

 Step 3

Doctor, start, then claim after epoch close

Use the dedicated v2 commands for the live Worldland path.
01

STEP 1

Clone the repo into the current user folder.

Open a fresh PowerShell window and run these commands exactly. On Windows PowerShell, use npm.cmd instead of npm so you do not hit the npm.ps1 execution-policy error. 

cd $env:USERPROFILE
git clone https://github.com/sinmb79/Koinara-node.git koinara-node
cd $env:USERPROFILE\koinara-node
npm.cmd install
If you already cloned to Desktop, move the folder to C:\Users\<your-user>\koinara-node before continuing.
02

STEP 2

Run setup and keep the default runtime paths.

The setup wizard writes node.config.json and .env.local. The current default runtime root is the user home folder, not the cloned repo. 

npm.cmd run setup
  • Shared manifest root: %USERPROFILE%\.koinara-node\network
  • Artifact root: %USERPROFILE%\.koinara-node\artifacts
  • State root: %USERPROFILE%\.koinara-node\state
  • Keep wallet keys in local files only. Do not commit them.
03

STEP 3

Check, start the OpenClaw-backed provider, then claim after the epoch closes.

npm.cmd run openclaw:connect
npm.cmd run provider:v2:openclaw:check
npm.cmd run provider:v2:openclaw:start

# in another PowerShell window if you also run a verifier
npm.cmd run verifier:v2:start

# claim only after the current epoch has closed
npm.cmd run provider:v2:openclaw:claim
npm.cmd run verifier:v2:claim

Provider path

Use the OpenClaw-backed provider commands if you want the local agent to generate the response content.

Provider + verifier

Use separate .env.provider.local and .env.verifier.local files and separate PowerShell windows.
